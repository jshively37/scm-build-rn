# Overview
This Python script walks through programmatically how to create remote networks in Strata Cloud Managed Prisma Access.

## Limitations:
* Bandwidth must already be allocated to the region.
* Each payload only has a subset of the available options. If you need other IPSEC, IKE, etc. options you'll have to add them.
* Only checks if the object already exists. It doesn't compare to see if there are any changes. i.e. if you change the `encrpytion_algoritm` in the IPsec profile it won't update. You'll need to manually update or delete the object from Prisma Access.

## Creating a service account in Prisma Access

You will need to create a service account in Prisma Access to generate the access token.

The Client ID, Secret ID, and TSG ID will be saved in the .env file.

To create a service account: https://pan.dev/sase/docs/service-accounts/


## Installing and running

Clone down this repo and cd into the directory.

Create a Python virtual environment: python -m venv venv

Activate the virtual environment: \
Windows: .\venv\Scripts\activate \
OSX: source venv/bin/activate \

Install the dependencies using `pip install -r requirements.txt`

Copy `env.example` to `.env` and populate your variables in this file generated while creating the service account. The .env is included in the .gitignore to ensure it does not accidentally get checked into a git repo.

A sample `remote_networks.yaml` is provided. If you need to any fields you will need to add the corresponding logic to `main.py`

Run: `python main.py`

You will be prompted for a number of domains to create.
